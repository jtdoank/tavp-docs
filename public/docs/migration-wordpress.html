# WordPress to TAVP Migration Guide > Complete guide for migrating from WordPress to TAVP Stack. ## Why Migrate? | Feature | WordPress | TAVP | |---------|-----------|------| | Performance | ~150 req/s | ~12,000 req/s | | Security | Frequent vulnerabilities | Secure by default | | Customization | Plugin-dependent | Full control | | Maintenance | High (updates, patches) | Low | | Scalability | Limited | Enterprise-ready | ## Migration Steps ### 1. Export WordPress Content ```bash # Export posts wp export --post_type=post --file=posts.xml # Export pages wp export --post_type=page --file=pages.xml # Export media wp export --post_type=attachment --file=media.xml # Export users wp export --post_type=user --file=users.xml ``` ### 2. Install TAVP ```bash composer create-project tavp/core my-app cd my-app tavp serve ``` ### 3. Create Database Schema ```php // database/migrations/001_create_posts_table.php use Tavp\Database\Schema\Blueprint; use Tavp\Support\Facades\Schema; Schema::create('posts', function (Blueprint $table) { $table->id(); $table->string('title'); $table->string('slug')->unique(); $table->text('content'); $table->string('status')->default('draft'); $table->timestamps(); }); ``` ### 4. Import Content ```php // Import posts from WordPress XML $xml = simplexml_load_file('posts.xml'); foreach ($xml->channel->item as $item) { Post::create([ 'title' => (string) $item->title, 'slug' => (string) $item->children('wp', true)->post_name, 'content' => (string) $item->children('content', true)->encoded, 'status' => (string) $item->children('wp', true)->status, ]); } ``` ### 5. Migrate Users ```php // Import users $xml = simplexml_load_file('users.xml'); foreach ($xml->channel->item as $item) { User::create([ 'name' => (string) $item->title, 'email' => (string) $item->children('wp', true)->user_email, 'password' => bcrypt('temporary-password'), ]); } ``` ### 6. Create CMS Structure ```php // config/cms.php return [ 'post_types' => [ 'post' => [ 'label' => 'Posts', 'supports' => ['title', 'editor', 'thumbnail', 'excerpt'], ], 'page' => [ 'label' => 'Pages', 'supports' => ['title', 'editor', 'thumbnail'], ], ], ]; ``` ### 7. Build Frontend ```php // resources/views/post.volt {% extends 'layouts/app.volt' %} {% block content %}

{{ post.title }}

{{ post.content }}
{% endblock %} ``` ### 8. Set Up Admin Panel ```bash # Install TAVPhub (admin panel) composer require tavp/hub tavp hub:install # Create post management tavp hub:make:resource Post ``` ## Data Migration Checklist - [ ] Posts (title, content, slug, status, dates) - [ ] Pages (title, content, slug, template) - [ ] Categories & Tags - [ ] Media (images, files) - [ ] Users (name, email, roles) - [ ] Comments - [ ] Menus - [ ] Widgets - [ ] Settings ## Performance Comparison | Metric | WordPress | TAVP | |--------|-----------|------| | Requests/sec | 150 | 12,000 | | Memory | 128MB | 8MB | | Page load | <500ms | <5ms | | Database queries | 20-50 | 1-3 | ## Need Help? - [TAVP Discord](https://discord.gg/tavp) - [GitHub Discussions](https://github.com/tavp-stack/tavp-core/discussions)
